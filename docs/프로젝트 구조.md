## 왜 이렇게 나누나?

### Figma 비유로 이해하기

```
Figma 파일 구성          →    코드 구성
──────────────                ─────────
📄 Pages                 →    app/ (페이지)
🧩 Components            →    components/ (재사용 블록)
🎨 Styles                →    globals.css (스타일)
📐 Variables/Tokens      →    types/ + data/ (설계도+데이터)
```

### 관심사의 분리 (Separation of Concerns)

한 파일이 한 가지 역할만 하도록 나눠요:
- **보여주는 것** (components) ≠ **계산하는 것** (lib) ≠ **기억하는 것** (store)

카드의 생김새를 바꾸고 싶으면 → `components/cards/` 만 수정
카드 뽑기 확률을 바꾸고 싶으면 → `lib/deck.ts`만 수정
새 카드를 추가하고 싶으면 → `data/cards.ts`만 수정

## 주요 파일 역할 요약

| 파일 | 역할 | 비유 |
|------|------|------|
| `types/card.ts` | 카드의 설계도 | Figma Component Properties |
| `data/cards.ts` | 실제 카드 내용 | Figma Instance들의 데이터 |
| `lib/deck.ts` | 카드 뽑기 로직 | 게임의 룰북 |
| `store/useGameStore.ts` | 앱 전체 상태 | 게임의 진행 기록 |
| `components/cards/ChallengeCard.tsx` | 카드 UI | Figma의 Card Component |
